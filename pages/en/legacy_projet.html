2017 Semester 1 Vocal recognition using deep learning (School project)

This project was managed by a group of three people. The goal was to discover AI and to make an application: voice recognition. We had two choices:

  • Take some existing code, plug’n’play like (easy to use, even if you do not understand it)
  • Do everything by ourselves from scratch (harder)

The big issue was that we did not know anything about AI. But we choose the second option, because it had also advantages, for example hading to learn some fundamentals about AI.

The project was not completed due to lack of time, but the software complied with the first part of the specifications.

Since we didn’t use any API for deep learning, the first part of our work was to self-learn basics of AI needed to realise this project.

Right after this self-learning, we started to build the software: neurons, then activation function (Sigmoid), back-propagation, and settings for the neural network. Each setting is saved in a specific text file (one file per layer with all neuron’s weight, and one master file with number of layers, and number of neurons for each layer)

The program operates as follows: an .mp3 file is sent to the software. Then converted from audio to an image (DSP) (only this part of the software is written in Python). Each pixel of the image is converted to 3 RGB values, and then injected in a neural network column after column. When a column is injected in the neural network, a forward propagation is triggered, and a letter is guessed by the software. Two possible cases are available then: the software can learn what was the correct letter using a backpropagation algorithm, or just print the guessed letter on screen.

I was in charge of the user interface (with Qt) and also general architecture of the software (based on MVC). I also helped implementation of backpropagation, and neuron’s structure.

This project has proved partially operational, but was not led to term due to the lack of time. However, it brought us knowledge and a constructive approach to AI.



2017 Semetre 1 Procedural map for a 2D game (School project, 3 people)

This project was done alongside the AI project. It consisted in creating a 2D game using only C++ and Qt.

I developed the map system and the associated environments: the map is ramdomly generated, but with a minimum of control to keep the game playable.

A video is showing the game in the technical part

The map is built from blocks: a bloc is a free or locked space. The map is square 120 blocks wide, and is cut in 16 areas. Each of those areas is also a square 30 blocks wide. An area represents an environment: desert, flower plain, jungle or city.

Area after area, the software will randomly choose one of the environment, and generate the 90 blocks inside. There are two types of blocks: the locked space (or obstacle), and the free space where the player can move. Each of those blocks is randomly chosen, however in order to make the game playable, the selection of blocks is weighted: there is more chance to have a free block.

When the area is filled, it is saved in a table that represent the entire map. And when this table is full, the map is sent to the game.

To understand which type of blocks the game should print on screen and its physic properties, each block is identified with a two-digit number:

The first number (or tens) indicates the environment, and the second number (or units) the type of block: from 0 to 6, it is a “free block” with different textures, 7, 8 and 9 are obstacles (rock, tree, pond, etc).

For example, 02 indicate a block of grass in a jungle, 09 a tree in a jungle. 12 a block of flowers in a plain, and 18 a rock a plain.

The map is generated before each party



2016 Semeter 2 Basic image editor (School project, single)

In 4th year first semester, I had to develop an image editor using Qt and threads properties.

The software allows to put multiple images on a background, to move them, rotate them right or left degree by degree, or grayscale them (using RGB). Images can be locked or unlocked to prevent modification.

Each transformation can be done with one or multiple selected images.

Grayscale is done using Qthreads.
